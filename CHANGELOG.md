### 2026-03-19

- **Chat no-document flow and approval gate fix**: Fixed chat behavior when no document is selected by introducing a dedicated no-context prompt path so the assistant no longer responds with document-lookup refusal patterns for direct user-provided text. Also fixed approval flow so `require_approval` is enforced before LLM calls even in no-document chats, removed temporary debug instrumentation added during runtime investigation, and aligned chat-context prompt tests with the updated no-context prompt wording to resolve CI failure.
- **Sanitization pipeline overhaul — fix broken masking and false positives**: Resolved systemic PII sanitization failures across NER, blocklist, and Ollama layers. Root causes: (1) NER LOC→LOCATION mapping produced massive false positives (city names, partial words tagged as locations despite LOCATION not being a regulation entity type); (2) subword tokenisation caused mid-word replacements (e.g. "An[LOCATION_1]No:81"); (3) blocklist propagated ALL entity tokens indiscriminately, corrupting entire documents with cascading replacements; (4) NER threshold too low (0.75) for non-English, allowing common words as PERSON_NAME; (5) short queries destroyed by NER + Ollama alias layer; (6) Ollama alias prompt overly broad. Fixes: removed LOC from NER label mapping (address detection delegated to regulation-specific Presidio recognizers); added word-boundary snapping for NER spans; raised NER confidence threshold to 0.85 for all languages; added minimum 3-char span length for NER; restricted blocklist to person-identifying entity types only (PERSON_NAME, ALIAS, etc.); skip NER for texts <50 chars and Ollama alias for texts <80 chars; made Ollama alias prompt strictly person-name-focused; filtered Ollama alias output to only accept person-identifying types; tightened `_expand_person_name_spans` to require name-like original span before expansion.
- **Architecture rule compliance sweep**: NER layer now filters results against the active policy's entity types (previously PERSON_NAME and EMAIL_ADDRESS were added unconditionally). Moved 3 inline LLM prompt fragments from `chat_context.py` and `chat.py` into `PromptCatalog` (`json_output_instruction`, `spreadsheet_schema_instruction`, `placeholder_list_instruction`). Moved `refusal_phrases` from `chat.py` into `PromptCatalog.REFUSAL_PHRASES` with FUTURE-to-DB comment. Replaced hardcoded `language="en"` in Presidio call with configurable `_PRESIDIO_DEFAULT_LANGUAGE` constant. Added explicit FUTURE comment to `SANITIZER_STOPWORDS`. Removed redundant code comments. Updated README (EN/TR) PII pipeline sections to reflect current NER behaviour, blocklist restrictions, and Ollama alias scope.
- **Critical: Ollama PII validation no longer strips structured identifiers**: The validation layer previously sent all spans (including `NATIONAL_ID`, `IBAN`, `PHONE_NUMBER`) to Ollama; an empty or `[]` JSON response caused `_ollama_validate_pii_candidates` to return an empty list, **dropping every detection including valid TCKNs** so raw IDs could reach the cloud LLM. Fix: `NATIONAL_ID`, `IBAN`, and related high-priority types bypass LLM validation entirely; empty or unparseable Ollama responses now fall back to keeping non-passthrough candidates instead of clearing them. Added `_merge_adjacent_person_name_spans` so consecutive `PERSON_NAME` spans separated only by spaces merge into one replacement, reducing broken mid-name placeholders.
